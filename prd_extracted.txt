PM Board V21
交互体验优化 · 产品需求文档
PRD Version 21.0  ·  2026-05-06
1. 文档概述
本文档描述 PM Board 系统 V21 迭代所需完成的全部交互优化需求，以及配套的数据库变更 SQL。
本次迭代聚焦于五类问题的修复与增强：
标签管理 — 弹窗状态保持，删除后不关闭弹窗
双刷新问题 — 多处写操作触发两次渲染，统一修复
甘特图无限滚动 — 内容区高度与实际行数对齐
甘特图筛选增强 — 新增项目 + 模块两级联动筛选
其他交互细节 — 发现并修复清单中列举的其余问题
2. 问题与需求全景清单
2.1 标签管理交互（FEAT-TAG）
现状：每次只能通过"系统配置→标签"入口打开标签管理弹窗；删除标签后 showConfirm 回调调用 closeModal()，弹窗关闭，用户需重新导航才能继续操作。
期望：
删除标签后不关闭弹窗，仅刷新标签列表区域
编辑标签名保存后同样仅刷新列表区域
添加标签成功后输入框清空，聚焦等待下一次输入，弹窗不关闭
弹窗支持通过侧边栏"标签"菜单项直接打开（已有入口，保持不变）
2.2 双刷新问题（FIX-DOUBLE-RENDER）
现状：多处写操作调用 render() 后，Supabase 实时订阅回调再次触发 loadState()→render()，产生两次渲染，用户会看到页面闪动或列表跳动。
已修复的函数（V17/V19）：toggleDone、toggleSubtask、confirmDeleteTask。
仍需修复的函数（V21 新增）：
pm-tasks.js: submitEditTask() — 调用 render() 前缺少 _lastLoadTime = Date.now()
pm-projects.js: submitEditProject()、confirmDeleteProject()、removeMemberFromProject()
pm-members.js: submitAddMember()、deleteMember() — 当前只刷新弹窗内 DOM，但实时回调仍触发全局 render()
pm-members.js: deleteTag() — showConfirm 确认后触发 deleteFromCloud，实时回调双刷新
💡 修复方式：在 syncTask/syncProject/deleteFromCloud 之后、render() 之前，统一插入 _lastLoadTime = Date.now()
2.3 甘特图底部无限滚动（FIX-GANTT-SCROLL）
现状：甘特图整体容器 .gantt-wrap 设置了 overflow-y:auto 但未限制高度，当内容行数较少时，浏览器仍可向下拖动滚动条，出现大量空白。
期望：
甘特图可滚动区域高度应精确匹配内容高度，不出现空白
.gantt-wrap 高度由内容决定，不超出视口可用高度（建议 max-height: calc(100vh - 220px)），超出后可滚动
左右列同步滚动已实现，不影响
2.4 甘特图项目+模块联动筛选（FEAT-GANTT-FILTER）
现状：甘特图工具栏已有成员筛选和状态筛选，但缺少按项目筛选；另外 V20 引入了 module_id（任务模块字段），也需要在甘特图中支持按模块筛选，且模块列表应随项目筛选联动。
新增筛选控件（插入在成员筛选之前）：
项目筛选下拉：「全部项目」+ 各项目名。选择某项目后，模块筛选下拉自动更新为该项目下的模块列表
模块筛选下拉：默认「全部模块」，仅在项目筛选选中某项目时启用；未选项目时置灰禁用
清除筛选按钮同步清空项目和模块筛选
数据来源：
项目列表：state.projects
模块列表：state.tasks 中 module_id 字段去重，或 modules 表（若存在）
💡 新增全局变量 window._ganttFilterProject = '' 和 window._ganttFilterModule = ''
2.5 其他交互细节修复（FIX-UX）
以下问题通过代码扫描和上下文分析识别：
UX-01  新建任务弹窗关闭后再次打开，多负责人 Chip 残留
现象：window._newTaskAssignees 是全局变量，关闭弹窗后不清空，再次打开新建任务弹窗时会显示上次选择的负责人。
修复：openAddTask() 开头重置 window._newTaskAssignees = []
UX-02  编辑任务弹窗多负责人 Chip 残留
现象：window._editTaskAssignees 在编辑弹窗打开前由 buildEditTaskAssigneeHTML() 初始化，但如果两次连续打开不同任务的编辑弹窗，旧数据可能残留。
修复：openEditTask() 开头显式重置 window._editTaskAssignees = []，确保 buildEditTaskAssigneeHTML() 始终从干净状态读取
UX-03  任务列表筛选条件切换后滚动位置未重置
现象：用户在任务列表底部切换项目/状态筛选芯片后，列表内容刷新但滚动条停留在底部，看起来列表为空。
修复：render() → renderTaskList() 渲染完成后调用 document.getElementById('main-content').scrollTop = 0
UX-04  成员管理弹窗删除成员后列表不刷新
现象：deleteMember() 调用 deleteFromCloud 成功后，仅在 state.members 中移除了该成员，但弹窗内的 members-list DOM 未重新渲染，用户看不到变化，需手动关闭再打开弹窗。
修复：在 state.members 过滤后，更新 document.getElementById('members-list').innerHTML = buildMembersListHTML()
UX-05  项目成员移除后弹窗刷新方式不一致
现象：removeMemberFromProject() 在 syncProject 成功后调用 openEditProject(projId) 重新打开整个弹窗，会导致弹窗动画重播和滚动位置丢失。
修复：改为仅刷新弹窗内成员列表区域，而不是重新 openModal
UX-06  甘特图折叠状态在缩放后丢失
现象：点击缩放按钮（大/中/小）调用 setGanttZoom(w) → renderGantt()，但 renderGantt() 会在 window._ganttCollapsed === null 时重新初始化折叠状态，导致每次缩放都折叠所有项目组。
修复：setGanttZoom() 中不重置 window._ganttCollapsed，保持当前折叠状态
💡 当前 renderGantt() 中的初始化逻辑：if (window._ganttCollapsed === null) { 初始化 }，setGanttZoom 只需不重置该变量即可
UX-07  Toast 通知与弹窗遮罩层叠问题
现象：在弹窗打开状态下触发 toast()，toast 元素被 .modal-overlay（z-index 通常高于 toast）遮盖，导致提示不可见。
修复：检查 pm.css 中 #toast-container 的 z-index（建议 99999），确保高于 .modal-overlay（z-index: 9000）
UX-08  周视图「+N 更多」点击无反应
现象：renderWeekGrid() 中每列最多显示 6 个任务，超出时显示「+N 更多」标签，但该标签没有绑定点击事件，用户无法查看完整列表。
修复：为 .week-more-label 添加 onclick，弹出当日所有任务的简单列表弹窗
UX-09  甘特图筛选清除按钮只清除状态/成员，不清除新增的项目/模块筛选
现象（FEAT-GANTT-FILTER 引入后）：清除筛选按钮 onclick 只重置 _ganttFilterStatus 和 _ganttFilterAssignee，不重置 _ganttFilterProject 和 _ganttFilterModule。
修复：清除筛选按钮同步重置四个筛选变量
UX-10  Finance 快捷录入弹窗金额字段缺少数字键盘提示
现象：快捷录入收款/支付弹窗中金额输入框 type="text"，在移动端不会弹出数字键盘；此外无 inputmode 属性。
修复：金额输入框添加 inputmode="decimal"，便于移动端输入

4. 数据库变更 SQL（V21_SQL.sql）
本次迭代均为前端交互修复，无新增数据表或字段。以下 SQL 为确认性检查语句，用于验证 V20 新增字段是否已在目标环境中存在，以及补全缺失的索引。
4.1 检查 V20 遗留字段
-- 确认 tasks.assignees 字段存在（V20 引入，V21 甘特图筛选依赖）
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'tasks' AND column_name IN ('assignees', 'module_id');
-- 若 assignees 不存在，执行：
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS assignees JSONB DEFAULT '[]'::jsonb;
-- 若 module_id 不存在，执行：
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS module_id TEXT;
4.2 确认 notifications 表存在（V19 引入）
-- 确认通知表存在
SELECT to_regclass('public.notifications');
-- 若返回 NULL，请执行 supabase/migrations/004_notifications.sql
4.3 添加甘特图筛选优化索引
-- 项目筛选查询优化
CREATE INDEX IF NOT EXISTS idx_tasks_project_id ON tasks(project_id);
-- 模块筛选查询优化
CREATE INDEX IF NOT EXISTS idx_tasks_module_id ON tasks(module_id);
-- 多负责人字段 GIN 索引（加速 assignees @> 查询）
CREATE INDEX IF NOT EXISTS idx_tasks_assignees_gin ON tasks USING GIN(assignees);
4.4 完整 V21_SQL.sql 文件内容
将以下内容保存为 sql/V21_SQL.sql：
-- V21 SQL：确认性检查 + 缺失字段补全 + 索引优化
-- 执行环境：Supabase SQL Editor
-- 执行时间：2026-05-06
-- Step 1: 检查 tasks 表必要字段
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS assignees  JSONB DEFAULT '[]'::jsonb;
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS module_id  TEXT;

SELECT column_name FROM information_schema.columns WHERE table_name='global_tags';

应对措施
_lastLoadTime 防抖失效
800ms 内多次操作时防抖保护失效

moduleId 字段命名
loadState() 中 module_id→moduleId 映射需确认

lucide.createIcons() 调用

在每次 innerHTML 赋值后调用 if(window.lucide)lucide.createIcons()
